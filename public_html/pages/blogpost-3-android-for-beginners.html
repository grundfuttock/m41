Android for Beginners

Configuring Android for accessibility or nervous users

Posted by Geoff on June 9, 2015

This is not a “Janet-and-John” introduction to Android. Android is hugely configurable. Here we highlight various configuration options that can be used to make an Android device (phone or tablet) easier to use for novices, those who lack confidence or for those with some form of impairment.

The use of Android launchers is covered in a separate article http://m41.co.uk/pages/blogpost-2-android--made-easy.html

Different users will have different requirements. A user with visual impairment will have different needs to one with poor dexterity. Equally they will have different needs to someone who just lacks the confidence to use a complex high tech device.  Not all of the options discussed below will be suitable for all of these users. Similarly, there are features within Android outside the “Accessibility” feature set that can be helpful to all these users.

Display

Orientation

Automatic change of orientation (landscape vs Portrait) can be confusing. Consider locking tablet in landscape orientation or disabling phone auto-rotation.

Font

Font sizes can be changed. Bear in mind that not all applications will honour the font size.

Background

A high contrast plain background makes life easier.

Home Screen(s)

Remove all the icons and widgets that are not needed. If possible, reduce the number of home screens such that there are no empty screens. If your selected launcher allows it, lock the setup so that icons cannot be moved or removed.

Depending on the user, it may not be necessary to have access to the apps drawer.

Input

Keyboard
Suggest a Bluetooth keyboard if the user will be doing any considerable amount of typing.
Stylus.
A stylus has proved very helpful for those who have difficulty typing on a touchscreen keyboard
Different keyboards
Android keyboards can be changed in software e.g. SwiftKey or fleksy.
Voice input
Google has a nifty voice recognition system for search, reminders, making calls etc. It is much easier to use than typing. Make use of it. In addition various manufacturers provide voice control for various functions on their devices. This varies from device to device, so check what is available on the device you are configuring.

Device Security

This is a tricky one. While it is always a good idea to have some form of security on a mobile device you have to consider how and where it is going to be used together with the inconvenience and / or confusion that security measures add. Some Android tablets have facial recognition available but this is not reliable and can cause more trouble than it’s worth. Setting a password or PIN provides security but can easily be forgotten.

A high proportion of customers for advice sessions that I have run have forgotten passwords with impacts ranging from having to set up a new account for some website through to having to send the device back to the supplier.

The level and type of security to be used very much depends on the user. WRITE IT DOWN!

The Android lock screen password/pin/pattern can be reset if the Android Device Manager App (ADM) has been installed on the device by logging-in to: google.com/android/devicemanager

Device / User Accounts

It can be awkward setting up accounts for the first time. This includes accounts for applications that the user wants to use such as Skype or Facebook as well as the Initial Google setups. To avoid this area of potential confusion and to reduce the impression of complexity, accounts should be setup before use.

Create a Google account with recovery options in case of lost password (see Device Security).

Install Android Device Manager (ADM) on the device.

Setup automatic backups.

Setup remote locate in case device (or user) gets lost.

If e-mail is required make sure that this has been setup correctly to read and send e-mail from and to the user’s preferred (or existing) e-mail account.

Apps

Only install or make available those that are needed. Hide others using the chosen launcher (if this feature is available) or Android Disable.

Setup any necessary accounts before use.

Configure all apps to be automatically updated. This ensures that any security updates will be installed. There may be some applications where this is not appropriate, but in general it is better to be up to date.

Discuss application requirements with users. It is probably better to start with a small number of apps until the user gets to grips with the basic use of the device. Once confidence grows other apps can be introduced.

Typical apps would include:

  • Internet browser
  • Camera
  • Photos
  • Contacts
  • E-mail
  • Skype
  • Music player
  • YouTube
  • Maps
  • Facebook

Other

Timeouts
Increase the delay before screen blanks. This will use more battery but makes the device more user-friendly. Also consider other timeouts.
Touch&hold delay
This is a Google accessibility setting that sets how quickly device registers that you are touching the screen. Set this to longer if the user has limited dexterity.
Android Talkback
This Android accessibility feature uses the device speakers to describe user actions and is intended for people with severe visual impairment. It can be set up to use alternative gestures too.

For more information on this feature see this Google accessibility article https://support.google.com/accessibility/android/?hl=en-GB#topic=6007234

There are many other areas where Android can be configured such as announcing notifications and calls or issuing reminders (occasional beep) if there is a reminder outstanding. Check the Android accessibility help pages and explore the options on the device that you are using to discover features that the device manufacturer may have provided.

 

Finally, Google’s accessibility help pages can be found here: https://support.google.com/accessibility/android/
